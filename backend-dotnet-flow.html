JWT Validation Logic
// 1. Read Authorization header var authHeader = ctx.Request.Headers["Authorization"].FirstOrDefault(); // 2. Extract token after "Bearer " if (authHeader?.StartsWith("Bearer ") == true) { var token = authHeader.Substring(7); // 3. Use JwtService to validate + extract claims var email = jwtService.ExtractEmail(token); var role = jwtService.ExtractRole(token); if (email != null && jwtService.IsTokenValid(token)) { // 4. Store in HttpContext.Items for controllers ctx.Items["UserEmail"] = email; ctx.Items["UserRole"] = role; } } await _next(ctx); // always continue (soft check)
‚öë .NET difference from Java: .NET stores user info in HttpContext.Items[]. Java uses SecurityContextHolder.setAuthentication(). Both achieve the same goal ‚Äî making user identity available downstream.

JwtService.cs ‚Äî GenerateToken()
public string GenerateToken(string email, string role) { var claims = new[] { new Claim(JwtRegisteredClaimNames.Sub, email), // sub = email new Claim("role", role), // custom claim new Claim(JwtRegisteredClaimNames.Iat, timestamp) }; // SAME SECRET KEY as Java! var key = new SymmetricSecurityKey( Encoding.UTF8.GetBytes("MySecretKeyForJwt123456789012345")); var creds = new SigningCredentials(key, HmacSha256); var token = new JwtSecurityToken( claims: claims, expires: DateTime.UtcNow.AddHours(24), // 24 hour expiry signingCredentials: creds); return new JwtSecurityTokenHandler().WriteToken(token); }
üîë Same secret key as Java backend: "MySecretKeyForJwt123456789012345" ‚Äî tokens are interoperable between both backends!
